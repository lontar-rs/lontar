# Journal Submission Systems Survey

## Overview

This document surveys major academic journal submission systems to determine which LaTeX engines they support. This is critical for Lontar's LaTeX backend to ensure compatibility with target journals.

## LaTeX Engine Support

### XeLaTeX vs pdfLaTeX

**XeLaTeX:**
- Unicode support (native UTF-8)
- System fonts via fontspec
- Required for multi-script documents
- Modern font features (OpenType)

**pdfLaTeX:**
- Legacy 8-bit encoding
- Limited Unicode support
- Font packages only
- Faster compilation

**LuaLaTeX:**
- Similar to XeLaTeX
- Lua scripting support
- Better for complex documents

## Major Publishers

### Elsevier

**Submission system:** Editorial Manager (EM)

**LaTeX support:**
- ✅ pdfLaTeX (primary)
- ✅ XeLaTeX (supported)
- ✅ LuaLaTeX (supported)
- Document class: `elsarticle.cls`

**Requirements:**
- Submit `.tex` source + `.bib` + figures
- PDF generated by system
- Accepts fontspec for XeLaTeX

**Notes:**
- Most authors use pdfLaTeX
- XeLaTeX accepted for multi-script content
- Must declare engine in manuscript

**Example:**
```latex
\documentclass{elsarticle}
% XeLaTeX compatible
\usepackage{fontspec}
\setmainfont{Times New Roman}
```

### Springer Nature

**Submission system:** Editorial Manager / SNAPP

**LaTeX support:**
- ✅ pdfLaTeX (primary)
- ✅ XeLaTeX (supported)
- ✅ LuaLaTeX (supported)
- Document class: `svjour3.cls`, `sn-jnl.cls`

**Requirements:**
- Submit `.tex` source + `.bib` + figures
- PDF for review
- Camera-ready uses publisher template

**Notes:**
- Flexible engine support
- XeLaTeX recommended for non-Latin scripts
- Good fontspec support

### Wiley

**Submission system:** ScholarOne Manuscripts

**LaTeX support:**
- ✅ pdfLaTeX (primary)
- ⚠️ XeLaTeX (limited support)
- ❌ LuaLaTeX (not officially supported)
- Document class: Journal-specific

**Requirements:**
- Submit PDF for review
- LaTeX source for production
- May require pdfLaTeX recompilation

**Notes:**
- Prefers pdfLaTeX
- XeLaTeX may require special handling
- Contact editor for multi-script content

### IEEE

**Submission system:** IEEE Manuscript Central

**LaTeX support:**
- ✅ pdfLaTeX (required)
- ❌ XeLaTeX (not supported)
- ❌ LuaLaTeX (not supported)
- Document class: `IEEEtran.cls`

**Requirements:**
- pdfLaTeX only
- Submit PDF + source
- Strict template compliance

**Notes:**
- **No XeLaTeX support**
- Use `inputenc` for limited Unicode
- Multi-script content problematic

**Workaround for Lontar:**
- Generate PDF directly (not LaTeX)
- Or use pdfLaTeX-compatible output

### ACM

**Submission system:** HotCRP / EasyChair

**LaTeX support:**
- ✅ pdfLaTeX (primary)
- ✅ XeLaTeX (supported)
- ✅ LuaLaTeX (supported)
- Document class: `acmart.cls`

**Requirements:**
- Submit PDF for review
- LaTeX source for camera-ready
- Flexible engine choice

**Notes:**
- Modern template (acmart v1.9+)
- Good XeLaTeX support
- Fontspec compatible

### AMS (American Mathematical Society)

**Submission system:** Editorial Manager

**LaTeX support:**
- ✅ pdfLaTeX (primary)
- ✅ XeLaTeX (supported)
- ✅ LuaLaTeX (supported)
- Document class: `amsart.cls`, `amsbook.cls`

**Requirements:**
- Submit `.tex` + `.bib` + figures
- PDF for review
- Flexible engine

**Notes:**
- Math-focused
- XeLaTeX for Unicode math
- Good multi-script support

### Medical Journals (JAMA, NEJM, Lancet, BMJ)

**Submission system:** Various (ScholarOne, EM)

**LaTeX support:**
- ⚠️ Limited LaTeX support
- Most require Word (.docx)
- PDF accepted for review

**Requirements:**
- Prefer Word manuscripts
- LaTeX may be accepted for final production
- Vancouver citation style required

**Notes:**
- **Lontar advantage:** Generate DOCX directly
- No LaTeX engine concerns
- Focus on DOCX output quality

### arXiv (Preprint Server)

**Submission system:** arXiv.org

**LaTeX support:**
- ✅ pdfLaTeX (primary)
- ✅ XeLaTeX (supported, requires declaration)
- ✅ LuaLaTeX (supported, requires declaration)

**Requirements:**
- Submit `.tex` + `.bib` + figures
- Declare engine in `00README.XXX`
- Auto-compilation

**Notes:**
- Flexible engine support
- Must declare XeLaTeX/LuaLaTeX
- Good for multi-script preprints

**Declaration:**
```
% 00README.XXX
% Use XeLaTeX for compilation
\pdfoutput=1
```

## Recommendations for Lontar

### LaTeX Backend Strategy

1. **Primary target:** XeLaTeX
   - Best multi-script support
   - Modern font handling
   - Supported by most publishers

2. **Fallback:** pdfLaTeX compatibility
   - For IEEE and strict journals
   - Use `inputenc` for limited Unicode
   - Warn users about script limitations

3. **Alternative:** Direct PDF generation
   - Bypass LaTeX entirely
   - Use `lontar-pdf` backend
   - No engine compatibility issues

### Implementation

```rust
pub enum LatexEngine {
    XeLaTeX,    // Default for multi-script
    LuaLaTeX,   // Alternative
    PdfLaTeX,   // Fallback for compatibility
}

impl Document {
    pub fn write_latex_with_engine(
        &self,
        path: &Path,
        engine: LatexEngine,
    ) -> Result<(), Error> {
        match engine {
            LatexEngine::XeLaTeX => {
                // Use fontspec, Unicode
                self.write_xelatex(path)
            }
            LatexEngine::LuaLaTeX => {
                // Similar to XeLaTeX
                self.write_lualatex(path)
            }
            LatexEngine::PdfLaTeX => {
                // Limited Unicode, warn user
                if self.has_complex_scripts() {
                    eprintln!("Warning: pdfLaTeX has limited multi-script support");
                    eprintln!("Consider using XeLaTeX or generating PDF directly");
                }
                self.write_pdflatex(path)
            }
        }
    }
}
```

### User Guidance

```rust
// Detect target journal and recommend engine
pub fn recommend_latex_engine(journal: &str) -> LatexEngine {
    match journal.to_lowercase().as_str() {
        "ieee" => LatexEngine::PdfLaTeX,
        "elsevier" | "springer" | "acm" | "ams" => LatexEngine::XeLaTeX,
        _ => LatexEngine::XeLaTeX,  // Default
    }
}
```

## Testing

Create test documents for each engine:

```bash
# XeLaTeX (multi-script)
xelatex test_multiscript.tex

# pdfLaTeX (Latin only)
pdflatex test_latin.tex

# LuaLaTeX (alternative)
lualatex test_multiscript.tex
```

## Conclusion

**Key findings:**
1. Most publishers support XeLaTeX (except IEEE)
2. Medical journals prefer Word (DOCX)
3. arXiv supports all engines
4. pdfLaTeX is legacy but still required by some

**Lontar strategy:**
- Default to XeLaTeX for LaTeX output
- Provide pdfLaTeX fallback with warnings
- Prioritize DOCX for medical journals
- Support direct PDF generation as alternative
